## Pyhton-Email-Notifer
# In this <a href="https://github.com/Phoenix-Nickname-Root/Pyhton-Email-Notifer">Project</a>, I am sharing my own work on email notifications regarding the status of a Linux server. This software logs the following metrics: On/Off/Crash, Further details on how the program works are publicly available and outlined in the instructions below

## 3. Notification of a server crash [e.g., due to a power outage]

Next, we’ll look at how you, as a regular user (preferably on Linux), can set up this software—specifically, the process of configuring the sender’s account (the server from which emails will be sent) and the recipient’s email address (the email address where reports generated by this software will be sent).

Setup is done in two steps:

# Step 1:
Here you must set up the sender’s account. This email notification software works with email accounts via the SMTP protocol, so you will need to (sender) you will need to create an app password through which the account will be used to send emails, and you will also need to know the SMTP server (such as `smtp.gmail.com` or `smtp.live.com`) for successful authentication and operation of the software

# Step 2:
If you followed the instructions correctly in Step 1, you can proceed to Step 2 and begin configuring the server where the application password was previously created. Let’s proceed to configure the `config.json` file by entering the necessary data:

```json
{
    “name”: “Startup_Off_Crash Register (Py)”,
    “user”: “Your_Sender_Mail@domain.com”,
    “password”: “App_Password_For_Sender_Mail”,
    “smtp_host”: “smtp.domain.com”,
    “smtp_port”: 465,
    “mail_reciver”: “your_Reciver_mail@domain.com”
}
```

### In the `name` field, specify the display name from which emails will be sent
### In the `user` field, specify the email address used to set up the app password, which will then be entered in the `password` field
### In the `smtp_host` field, enter your email provider’s SMTP host domain (such as `smtp.gmail.com` or `smtp.live.com`)
### In the `smtp_port` field, enter the port for the SMTP protocol (this is necessary for the software to function correctly)
### In the `mail_reciver` field, enter your email address where you will receive notifications about the server being turned on, turned off, or crashing
